--- app/database.py ---
  - SQLite at data/mood_engine.db
  - init_db() creates tables + runs idempotent ALTER TABLE migration to add genres column
  - get_session() returns a plain SessionLocal()

--- app/auth.py ---
  - SpotifyOAuth manager built from SPOTIPY_CLIENT_ID / _CLIENT_SECRET / _REDIRECT_URI env vars
  - Scopes: user-library-read, user-top-read, user-read-recently-played,
            playlist-modify-public/private, user-read-private, user-read-email
  - get_spotify_client(): reads cached token, refreshes if expired
  - get_auth_url(), exchange_code(), get_current_user() helpers

--- app/fetch.py ---
  - Module-level _artist_genre_cache dict for process-lifetime artist→genres memoization
  - upsert_user(): fetches Spotify profile; email stored as profile.get("email") or ""
  - _upsert_track(): inserts Track, then fills genres via sp.artist() on first encounter
  - fetch_saved_tracks(): pages all Liked Songs via sp.next()
  - fetch_top_tracks(): all three time ranges (short/medium/long_term), limit=50 each
  - fetch_recently_played(): up to 50 recent plays; ListeningHistory upserted via
                             sqlite_insert().on_conflict_do_nothing()
  - sync_all(): upsert_user → fetch_saved_tracks → fetch_top_tracks → fetch_recently_played
  - load_tracks_dataframe(): simple Track query; returns track_id, spotify_track_id, name,
                             artist, album, popularity (0 if None), genres ("" if None)
  - load_history_dataframe(): LEFT JOIN ListeningHistory×Track; adds hour column

--- app/cluster.py ---
  - GENRE_TOP_N=50, K_MIN=2, K_MAX=10
  - _GENRE_MOOD_RULES: 34 ordered (genre-substring → mood-label) rules; specific
                       multi-word rules precede component words (e.g. "indie pop" before "pop")
  - build_feature_matrix(): top-N genre one-hot + NORMALIZED POPULARITY column →
                            StandardScaler → returns (X_scaled, scaler, feature_names)
  - select_k(): silhouette scoring across K_MIN…K_MAX; returns best K
  - run_kmeans(): fits KMeans with n_init="auto"
  - assign_clusters(): adds "cluster" int column to df copy
  - infer_mood_label(): first-match substring scan of top-genre list against rules
  - save_clusters(): deletes existing rows, inserts new Cluster rows with label only
                     (centroid audio columns left NULL)
  - load_cluster_labels(): returns {cluster_index: label} dict
  - infer_current_mood(): ±2-hour window of listening history → mode cluster
  - pca_projection(): 2D PCA for scatter visualisation
  - genre_distribution(): top-N genres per cluster by track count

--- app/recommend.py ---
  - get_library_tracks_for_cluster(): sorts cluster tracks by popularity DESC, returns
                                      spotify_track_ids up to limit=30
  - get_catalog_recommendations(): sp.recommendations(seed_tracks=seeds[:5], limit=limit);
                                   catches ANY exception and returns [] with a warning log
  - build_playlist(): library tracks first; if expand_catalog=True and under target_size,
                      fills remainder from catalog; deduplicates; caps at target_size
  - create_or_replace_playlist(): searches user's existing playlists for "Mood Engine: {label}";
                                  creates if absent; replaces all tracks in 100-item batches

--- app/ui.py ---
  - _handle_oauth(): reads ?code= query param, calls exchange_code(), clears params
  - _try_load_client(): loads cached Spotipy client into st.session_state.sp
  - _load_app_state(): queries DB for User by spotify_id; loads tracks DF + cluster labels
  - render_login(): title + link_button to Spotify OAuth URL
  - render_sidebar(): Sync Library + Re-cluster buttons; Current Mood metric
  - render_charts(): PCA scatter (Plotly) + genre bar chart per cluster
  - render_playlist_section(): mood selector (deduplicates same-label clusters), track count
                               input, catalog expansion checkbox, "Create Playlist" button
  - main(): set_page_config → init_db → handle OAuth → load client → render all sections
